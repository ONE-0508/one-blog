## 1. 项目背景与目标

### 1.1 背景

当前博客项目：

- 前端首页文章区域仍为占位
- 后端仅有 auth 路由，文章路由未接入
- 架构文档已规划 posts 模块与 `/admin` 路由

### 1.2 问题陈述

**当前痛点**：

- 管理员无法在后台管理文章内容
- 前台无法展示真实文章内容，仅为占位
- 文章数据缺少统一的增删改查入口

**现有方案不足**：

- 仅有认证相关路由，文章模块未落地
- 无管理端入口，无法进行内容生产与维护

### 1.3 目标

- 支持文章在后台管理端的新增/编辑/删除
- 前台读者端展示文章列表与详情
- 后端提供文章 CRUD API

### 1.4 成功指标（KPI）

- 管理端完成新增/编辑/删除后，前端列表与详情实时可见更新
- 文章列表按时间倒序正确展示
- 前端列表分页正常

### 1.5 业务目标与约束

**业务目标**：

- 短期（本期迭代）：完成文章管理闭环与前台展示
- 中期（后续迭代）：引入更丰富内容编辑与运营能力
- 长期（远期）：形成可扩展内容体系（分类、标签、专题）

**已知约束**：

- 管理端为单人使用
- 管理端使用 React 模板，不涉及 SSR
- 不支持草稿、定时发布
- 文章唯一标识使用 `id`
- 删除采用软删除

### 1.6 范围与约束

**范围**：

- 前端读者端（列表、详情）
- 独立管理端（文章管理）
- 后端 API/数据

**明确不做**：

- 草稿、定时发布
- 文章标识字段（slug 等）
- 条件筛选与搜索（本期不做）

**约束**：

- 管理端目录：`/Users/mac/code/blog/admin`

## 2. 目标用户与场景

### 2.1 目标用户

- 博客管理员（单人）
- 博客读者（匿名访问）

### 2.2 关键场景

1. 管理员在后台新建文章并发布，前台列表与详情可见
2. 管理员编辑文章内容，前台详情更新
3. 管理员删除文章，前台列表与详情不可见（软删除）

## 3. 功能需求

### 3.1 功能架构图

```
文章管理系统
├── 读者端
│   ├── 文章列表
│   ├── 文章详情
│   └── 分页展示
├── 管理端（独立）
│   ├── 文章列表管理
│   ├── 新建文章
│   ├── 编辑文章
│   └── 删除文章（软删除）
└── 后端服务
    ├── 文章 CRUD API
    └── 文章数据存储
```

### 3.2 文章数据字段

最少字段：

- `id`
- `title`
- `content`
- `tags`
- `author`
- `createTime`
- `updateTime`
- `viewCount`（阅读量，需持久化）

### 3.4 功能详细说明

#### 3.4.1 后端（文章 API）

**资源**：`articles`

- `POST /api/v1/articles` 新增文章
- `PUT /api/v1/articles/:id` 编辑文章
- `DELETE /api/v1/articles/:id` 软删除文章
- `GET /api/v1/articles` 文章列表（支持分页、按时间倒序）
- `GET /api/v1/articles/:id` 文章详情

**列表参数**：

- `page`（页码）
- `pageSize`（每页数量）
- `sort`（默认 `createTime desc`）

**软删除规则**：

- 删除后前台与列表不可见
- 管理端默认不展示已删除文章（除非后续新增回收站需求）

#### 3.4.2 前台读者端

**文章列表页**：

- 按时间倒序展示
- 展示字段：标题、作者、发布时间、阅读量
- 目录（TOC）
- 分页

**文章详情页**：

- 展示标题、作者、发布时间、正文内容
- 目录（TOC）
- 阅读量

#### 3.4.3 管理端（独立）

**入口**：独立管理端项目，目录为 `/admin`

**功能模块**：

- 文章列表（支持分页、按时间倒序）
- 新建文章
- 编辑文章
- 删除文章（软删除）

**权限**：

- 单人使用，暂不做多账号权限体系

**编辑器**：

- 富文本 + Markdown（双支持）
