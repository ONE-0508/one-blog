# 开发指南

## 文档概述

本文档为后端项目开发人员提供详细的开发规范、工作流程和最佳实践指南。

## 1. 开发环境配置

### 1.1 系统要求
- Node.js 18+ (推荐使用nvm管理版本)
- npm 9+ 或 pnpm 8+
- Git 2.30+
- 数据库: PostgreSQL 14+ (可选，开发初期可使用SQLite)

### 1.2 环境设置步骤

#### 步骤1: 克隆项目
```bash
git clone <repository-url>
cd blog/backend
```

#### 步骤2: 安装依赖
```bash
# 使用npm
npm install

# 或使用pnpm（推荐）
pnpm install
```

#### 步骤3: 环境配置
```bash
# 复制环境变量模板

vim .env
```

```bash
# 开发模式（热重载）
npm run dev

# 或直接使用
node --loader ts-node/esm src/server.ts
```

## 2. 代码规范

### 2.1 命名约定

#### 文件命名
- 使用小写字母和连字符：`user-controller.ts`
- 类文件使用PascalCase：`UserController.ts`
- 测试文件：`user.controller.test.ts`
- 类型定义：`user.types.ts`

#### 变量命名
```typescript
// 常量 - 全大写，下划线分隔
const MAX_RETRY_COUNT = 3;
const API_BASE_URL = 'https://api.example.com';

// 变量 - camelCase
let userName = 'John';
const articleCount = 10;

// 布尔值 - 使用is/has/can/should前缀
const isActive = true;
const hasPermission = false;
const canEdit = true;

// 函数 - camelCase，动词开头
function getUserById(id: string) {}
async function fetchArticles() {}
function validateEmail(email: string) {}

// 类 - PascalCase
class UserService {}
interface ArticleRepository {}
type PaginationOptions = {}
```

#### 数据库命名
- 表名：复数，小写，下划线分隔：`users`, `article_comments`
- 列名：小写，下划线分隔：`created_at`, `updated_at`
- 外键：`user_id`, `article_id`
- 索引：`idx_table_column`, `uk_table_column`

### 2.2 代码格式

#### 缩进与空格
- 使用2个空格缩进
- 操作符前后加空格
- 逗号后加空格
- 花括号前加空格

```typescript
// 正确
function calculateSum(a: number, b: number): number {
  return a + b;
}

// 错误
function calculateSum(a:number,b:number):number{
  return a+b;
}
```

#### 行长度
- 最大行长度：100字符
- 超过时适当换行

```typescript
// 正确换行
const result = await userRepository.findMany({
  where: { isActive: true },
  include: { articles: true },
  orderBy: { createdAt: 'desc' },
});

// 函数参数换行
async function createUser(
  username: string,
  email: string,
  password: string,
  options?: CreateUserOptions
): Promise<User> {
  // ...
}
```

### 2.3 导入导出规范

#### 导入顺序
1. 第三方库
2. 内部模块
3. 类型导入
4. 样式/资源

```typescript
// 1. 第三方库
import express from 'express';
import { Request, Response } from 'express';

// 2. 内部模块
import UserService from '@/services/user.service';
import { validateUser } from '@/validators/user.validator';

// 3. 类型导入

// 4. 样式/资源（前端）
// import './user.css';
```

#### 导出规范
- 默认导出：每个文件一个默认导出
- 命名导出：相关功能分组导出
- 避免混合导出

```typescript
// 正确 - 默认导出
export default class UserController {
  // ...
}

// 正确 - 命名导出
export { UserService, UserRepository };
export type { User, CreateUserDto };

// 避免 - 混合导出
export default UserController;
export { UserService }; // 不好
```

## 3. TypeScript最佳实践

### 3.1 类型定义

#### 避免使用any
```typescript
// 不好
function processData(data: any) {
  return data.value;
}

// 好
interface Data {
  value: string;
  count: number;
}

function processData(data: Data) {
  return data.value;
}

// 如果确实需要灵活类型，使用unknown
function safeParse(json: string): unknown {
  return JSON.parse(json);
}
```

#### 使用类型别名和接口
```typescript
// 类型别名 - 用于联合类型、元组等
type UserRole = 'admin' | 'editor' | 'user' | 'guest';
type Coordinates = [number, number];
type Nullable<T> = T | null;

// 接口 - 用于对象形状
interface User {
  id: string;
  username: string;
  email: string;
  createdAt: Date;
}

// 扩展接口
interface AdminUser extends User {
  role: 'admin';
  permissions: string[];
}
```

#### 使用泛型
```typescript
// 泛型函数
function identity<T>(value: T): T {
  return value;
}

// 泛型接口
interface ApiResponse<T> {
  success: boolean;
  data: T;
  meta?: PaginationMeta;
}

// 泛型类
class Repository<T> {
  private items: T[] = [];
  
  add(item: T): void {
    this.items.push(item);
  }
  
  findById(id: string): T | undefined {
    return this.items.find(item => (item as any).id === id);
  }
}
```

### 3.2 错误处理

#### 使用自定义错误类
```typescript
// 定义错误类
export class AppError extends Error {
  constructor(
    message: string,
    public statusCode: number = 500,
    public isOperational: boolean = true
  ) {
    super(message);
    Error.captureStackTrace(this, this.constructor);
  }
}

// 使用
async function getUser(id: string) {
  const user = await userRepository.findById(id);
  if (!user) {
    throw new AppError('User not found', 404);
  }
  return user;
}
```

#### 异步错误处理
```typescript
// 使用try-catch
async function fetchData() {
  try {
    const data = await api.get('/data');
    return processData(data);
  } catch (error) {
    // 处理已知错误
    if (error instanceof ApiError) {
      logger.error('API error:', error);
      throw new AppError('Failed to fetch data', 502);
    }
    // 重新抛出未知错误
    throw error;
  }
}

// 避免嵌套的try-catch
// 不好
async function badExample() {
  try {
    const user = await getUser();
    try {
      const profile = await getProfile(user.id);
      // ...
    } catch (error) {
      // ...
    }
  } catch (error) {
    // ...
  }
}

// 好
async function goodExample() {
  const user = await getUser().catch(handleUserError);
  const profile = await getProfile(user.id).catch(handleProfileError);
  // ...
}
```

### 3.3 异步编程

#### 使用async/await
```typescript
// 好 - 使用async/await
async function getUserWithPosts(userId: string) {
  const user = await userRepository.findById(userId);
  const posts = await postRepository.findByUserId(userId);
  return { user, posts };
}

// 避免 - 回调地狱
function badExample(userId: string, callback: Function) {
  userRepository.findById(userId, (err, user) => {
    if (err) return callback(err);
    postRepository.findByUserId(userId, (err, posts) => {
      if (err) return callback(err);
      callback(null, { user, posts });
    });
  });
}
```

#### 并行处理
```typescript
// 顺序执行（慢）
async function sequential() {
  const user = await getUser();
  const posts = await getPosts();
  const comments = await getComments();
  return { user, posts, comments };
}

// 并行执行（快）
async function parallel() {
  const [user, posts, comments] = await Promise.all([
    getUser(),
    getPosts(),
    getComments(),
  ]);
  return { user, posts, comments };
}

// 部分并行
async function partialParallel() {
  // 先获取用户
  const user = await getUser();
  
  // 然后并行获取帖子和评论
  const [posts, comments] = await Promise.all([
    getPosts(user.id),
    getComments(user.id),
  ]);
  
  return { user, posts, comments };
}
```

```typescript
import { UserService } from '@/services/user.service';
import { UserRepository } from '@/repositories/user.repository';
import { ValidationError } from '@/utils/AppError';

// Mock依赖
jest.mock('@/repositories/user.repository');

describe('UserService', () => {
  let userService: UserService;
  let userRepository: jest.Mocked<UserRepository>;

  beforeEach(() => {
    userRepository = new UserRepository() as jest.Mocked<UserRepository>;
    userService = new UserService(userRepository);
  });

  describe('createUser', () => {
    it('should create user with valid data', async () => {
      // Arrange
      const userData = {
        username: 'testuser',
        email: 'test@example.com',
        password: 'Password123!',
      };
      
      userRepository.create.mockResolvedValue({
        id: 'uuid',
        ...userData,
        createdAt: new Date(),
      });

      // Act
      const result = await userService.createUser(userData);

      // Assert
      expect(userRepository.create).toHaveBeenCalledWith(
        expect.objectContaining({
          username: 'testuser',
          email: 'test@example.com',
        })
      );
      expect(result).toHaveProperty('id');
      expect(result).toHaveProperty('username', 'testuser');
    });

    it('should throw ValidationError for duplicate email', async () => {
      // Arrange
      const userData = {
        username: 'testuser',
        email: 'duplicate@example.com',
        password: 'Password123!',
      };
      
      userRepository.findByEmail.mockResolvedValue({
        id: 'existing-uuid',
        email: 'duplicate@example.com',
        username: 'existinguser',
        createdAt: new Date(),
      });

      // Act & Assert
      await expect(userService.createUser(userData))
        .rejects
        .toThrow(ValidationError);
    });
  });
});
```

### 4.3 测试最佳实践

#### 测试原则
- **FIRST原则**:
  - Fast: 测试要快
  - Independent: 测试要独立

  - Self-validating: 测试自验证
  - Timely: 及时编写测试
