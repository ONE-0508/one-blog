## 1. 概述

- **文档目的**：将《个人博客网站 PRD》中定义的功能（笔记记录、博客文章、留言板块、作品展示、数据统计、移动端适配）映射为具体的前端页面、路由与主要交互，为开发与联调提供页面级设计说明。
- **前端技术栈**：
  - 框架：React 19（函数组件 + Hooks）
  - 构建工具：Vite
  - 语言：TypeScript
  - 样式：TailwindCSS 4（原子类 + 自定义主题变量）
- **整体信息架构**：
  - 前台（读者端）：首页、文章详情、笔记列表/详情、分类/标签、归档、留言板、作品展示/详情、数据统计（可选公开）、关于、友链、搜索等。
  - 后台（作者端）：PRD 说明为“后台 UI 可套用框架，后端需要手写”，本设计文档聚焦当前仓库内的前台页面；后台如在本仓实现时，可参考 PRD 的 `/admin` 结构扩展。
- **设计原则**：
  - 移动端优先：在宽度 < 768px 情况下保证阅读与操作体验优先，桌面端增强布局（侧边栏、TOC 固定等）。
  - 主题支持：统一提供明暗主题切换，页面布局与组件需兼容两套主题。

## 2. 路由与页面总览

### 2.1 路由表

| 路径 | 页面名称 | 模块 | 说明 |
|------|----------|------|------|
| `/` | 首页 | 博客文章/推荐内容 | 文章为主的内容入口，可展示精选作品/最新笔记 |
| `/post/:slug` 或 `/p/:id` | 文章详情页 | 博客文章 | 展示单篇文章正文、TOC、评论等 |
| `/notes` | 笔记列表页 | 笔记记录 | 时间线或列表形式展示笔记 |
| `/notes/:id`（可选） | 笔记详情页 | 笔记记录 | 针对较长或需分享的笔记提供单独详情 |
| `/category/:name` | 分类列表页 | 内容导航 | 按分类聚合文章列表 |
| `/tag/:name` | 标签列表页 | 内容导航 | 按标签聚合文章列表 |
| `/archives` | 归档页 | 内容导航 | 按时间维度汇总文章 |
| `/guestbook` 或 `/message` | 留言板页 | 留言板块 | 读者留言与展示 |
| `/works` 或 `/projects` | 作品展示页 | 作品展示 | 以卡片形式展示多个作品 |
| `/works/:slug`（可选） | 作品详情页 | 作品展示 | 展示单个作品的详细介绍与链接 |
| `/stats`（可选） | 数据统计页 | 数据统计 | 展示访问趋势、热门内容等，如对外公开 |
| `/about` | 关于页 | 站点信息 | 展示博主信息、站点说明等 |
| `/links`（可选） | 友链页 | 扩展 | 展示友链列表 |
| `/search` 或搜索弹窗 | 搜索页/搜索结果 | 搜索 | 文章/笔记搜索与结果展示 |

> 说明：以上路径是推荐方案，实际实现时可根据路由库（如 React Router）进行调整，但需保持语义清晰、与 PRD 对齐。

## 3. 前台页面设计

本章按路由划分，描述每个页面的布局区域、功能点和建议组件划分。

### 3.1 首页 `/`

- **主要目标**：成为访问入口，突出最新/重要内容，提供到各模块（文章、笔记、作品等）的导航。
- **布局结构**：
  - 顶部导航栏（Header）：
    - 元素：站点 Logo/名称、主导航菜单（首页、笔记、作品、归档、关于、留言板等）、搜索入口（图标或输入框）、主题切换按钮。
    - 移动端：Logo + 汉堡菜单，展开后展示导航列表与主题切换。
  - 主内容区域：
    - 文章列表区：按时间倒序展示文章卡片。
    - 可选推荐区：顶部/中部插入“置顶文章”“推荐作品”等模块。
  - 侧边栏（仅桌面端展示）：
    - 热门文章列表（标题 + 简短信息）。
    - 分类/标签云（用于跳转）。
  - 页脚（Footer）：
    - 版权信息、备案信息（如有）、RSS 链接、社交链接。
- **主要功能点**：
  - 文章列表分页或“加载更多”（下拉或按钮）。
  - 点击文章卡片跳转到文章详情页。
  - 导航交互：点击导航菜单跳转到各模块页面。
  - 主题切换：切换明暗模式，持久化用户选择（localStorage）。
- **建议组件**：
  - 页面：`HomePage`
  - 区域：`PostListSection`、`SidebarSection`
  - UI：`PostCard`、`SectionTitle`

### 3.2 文章详情页 `/post/:slug` 或 `/p/:id`

- **主要目标**：提供舒适的阅读体验，强化目录导航和互动能力。
- **布局结构**：
  - 顶部导航栏：与首页一致。
  - 主内容区域：
    - 标题区：文章标题、副标题（可选）、发布时间、分类/标签、阅读数（如有）。
    - 正文区：Markdown 渲染后的内容，包含代码高亮。
    - 底部导航：上一篇/下一篇文章链接。
    - 评论区/评论占位：根据是否接入评论系统展示实际评论或“评论功能即将上线/使用某评论系统”的说明。
  - 侧边栏（桌面端）：
    - TOC（目录）：根据标题层级自动生成列表，当前阅读段落高亮。
  - 移动端：
    - TOC 抽屉：通过按钮打开/关闭目录。
- **主要功能点**：
  - 加载状态：首次进入详情页时展示骨架屏或 loading 提示。
  - 错误与 404：文章不存在时，展示“内容未找到”页面/组件。
  - TOC 同步滚动：滚动正文时高亮当前小节。
  - 分享：提供“复制链接”或分享按钮（可选）。
- **建议组件**：
  - 页面：`PostDetailPage`
  - 区域：`PostHeader`、`PostContent`、`PostFooterNav`、`CommentSection`、`PostToc`
  - UI：`CodeBlock`、`TagBadge`、`CategoryLink`

### 3.3 笔记列表页 `/notes`

- **主要目标**：快速浏览和回顾笔记/碎片化内容。
- **布局结构**：
  - 顶部导航：与首页共用。
  - 主内容区域：
    - 笔记列表：推荐按时间线或简洁卡片展示。
    - 顶部筛选栏（可选）：标签筛选、搜索框。
  - 页脚：共用。
- **主要功能点**：
  - 列表展示：每条笔记显示创建时间、标题（可选）或前几行内容、标签。
  - 分页/加载更多：支持按页加载或“查看更多”。
  - 过滤：按标签/关键字过滤（可作为后续迭代）。
  - 详情跳转（如有 `/notes/:id`）：点击卡片进入详细页。
- **建议组件**：
  - 页面：`NotesPage`
  - 区域：`NotesFilterBar`、`NoteList`
  - UI：`NoteCard`、`TagFilter`

### 3.4 笔记详情页 `/notes/:id`（可选）

- **主要目标**：为较长或独立分享的笔记提供完整阅读体验。
- **布局结构**：
  - 结构可复用文章详情页简化版：
    - 标题区（如有）、时间。
    - 正文 Markdown 渲染，支持代码高亮。
    - 可选评论占位。
- **主要功能点**：
  - 类似文章详情的加载、错误处理。
  - 链接复制分享。
- **建议组件**：
  - 页面：`NoteDetailPage`
  - 区域/UI：复用 `PostContent`、`CommentSection` 等组件。

### 3.5 分类页 `/category/:name` 与标签页 `/tag/:name`

- **主要目标**：为读者提供按主题浏览内容的入口。
- **布局结构**：
  - 顶部导航共用。
  - 主内容：
    - 标题栏：当前分类/标签名称、文章数量（可选）。
    - 文章列表：复用首页的文章卡片与分页。
  - 侧边栏（可选）：展示其他分类/标签列表。
- **主要功能点**：
  - 列表加载/分页。
  - 空状态：当前分类/标签下无文章时显示提示。
- **建议组件**：
  - 页面：`CategoryPage`、`TagPage`
  - 区域：`CategoryHeader`、`TagHeader`
  - UI：`PostCard`、`EmptyState`

### 3.6 归档页 `/archives`

- **主要目标**：按时间维度帮助读者系统回顾历史内容。
- **布局结构**：
  - 主内容：
    - 年份分组：按年份分组标题。
    - 月份列表：在年份下按月份展示文章条目列表（日期 + 标题）。
    - 可折叠：可折叠/展开年或月。
- **主要功能点**：
  - 折叠/展开交互。
  - 点击文章条目跳转详情。
- **建议组件**：
  - 页面：`ArchivesPage`
  - UI：`ArchiveYearSection`、`ArchiveMonthSection`

### 3.7 留言板页 `/guestbook` 或 `/message`

- **主要目标**：让读者可留言、反馈、互动。
- **布局结构**：
  - 介绍区：简短说明留言用途与规则（文明发言、频率限制等）。
  - 留言表单：
    - 字段：昵称（必填或可选）、邮箱（可选）、内容（必填，多行文本框）。
    - 提交按钮。
  - 留言列表：
    - 展示留言者昵称、时间、内容。
    - 可选回复内容（如后台支持回复）。
- **主要功能点**：
  - 表单校验：必填校验、长度限制，错误文案提示。
  - 提交状态：提交中 loading、成功提示（清空表单/保留）、失败错误提示。
  - 防刷配合：前端可以基于后端返回的频率限制错误码展示相应提示。
  - 列表分页/加载更多。
- **建议组件**：
  - 页面：`GuestbookPage`
  - 区域：`GuestbookForm`、`GuestbookList`
  - UI：`GuestbookItem`、表单基础组件（输入框、文本域、按钮）

### 3.8 作品展示页 `/works` 或 `/projects`

- **主要目标**：以视觉友好的方式展示博主的项目与作品。
- **布局结构**：
  - 顶部简介：简单介绍作品板块的定位（个人项目、开源项目等）。
  - 作品卡片栅格：
    - 卡片信息：封面图、标题、一句话简介、技术栈标签、时间（可选）、查看详情/访问链接按钮。
- **主要功能点**：
  - 筛选：按标签/类型（如“开源”“个人项目”“工作项目”）过滤。
  - 点击卡片：
    - 若有详情页 `/works/:slug`：进入详情。
    - 若仅外链：在新窗口打开外部链接（按钮文案清晰，如“查看 GitHub”、“访问 Demo”）。
- **建议组件**：
  - 页面：`WorksPage`
  - 区域：`WorkFilterBar`、`WorkGrid`
  - UI：`WorkCard`、`TagFilter`

### 3.9 作品详情页 `/works/:slug`（可选）

- **主要目标**：展示某个作品的详细背景、过程与成果。
- **布局结构**：
  - 标题与基础信息：项目名称、时间、角色（如个人/团队）、技术栈。
  - 内容区：图文介绍（问题背景、解决方案、亮点等）、截图/视频展示区。
  - 链接区：GitHub、在线 Demo、相关文章等多个链接按钮。
- **主要功能点**：
  - 媒体展示：图片画廊/轮播（可后续迭代）。
  - 快速访问链接。
- **建议组件**：
  - 页面：`WorkDetailPage`
  - 区域：`WorkOverview`、`WorkMediaGallery`、`WorkLinks`

### 3.10 数据统计页 `/stats`（可选公开）

- **主要目标**：以可视化的形式展示博客整体运行情况。
- **布局结构**：
  - 概览卡片区：
    - 指标卡：总文章数、总笔记数、总访问量（PV）、总留言数、总作品数等。
  - 趋势区：
    - 折线图/柱状图占位：按天/周访问量趋势。
  - 热门内容区：
    - 热门文章/热门笔记列表。
  - 空状态：
    - 数据不足时展示友好提示。
- **主要功能点**：
  - 时间范围切换（如近 7 天、近 30 天）可作为后续迭代。
  - loading/错误/空数据状态处理。
- **建议组件**：
  - 页面：`StatsPage`
  - 区域：`StatsOverviewCards`、`StatsTrendSection`、`StatsTopList`
  - UI：`StatCard`

### 3.11 关于页 `/about`

- **主要目标**：介绍博主本人和站点背景。
- **布局结构**：
  - 头像与基本信息。
  - 自我介绍文案。
  - 社交链接（GitHub、微博、Twitter、LinkedIn 等）。
  - 站点说明（技术栈、建站初衷等）。
- **建议组件**：
  - 页面：`AboutPage`
  - UI：`Avatar`、`SocialLinkList`

### 3.12 友链页 `/links`（可选）

- **布局结构**：
  - 简介文案：说明申请友链的要求（若需要）。
  - 友链列表：
    - 卡片包含：站点名称、简介、链接。
- **建议组件**：
  - 页面：`LinksPage`
  - UI：`LinkCard`

### 3.13 搜索页 `/search` 或搜索弹窗

- **主要目标**：帮助用户快速查找文章和笔记。
- **布局结构**：
  - 搜索输入区：输入框 + 搜索按钮（或即时搜索）。
  - 结果列表：
    - 每项展示：标题、摘要、发布日期、类型（文章/笔记）、匹配关键词高亮（可后续迭代）。
  - 无结果提示区。
- **主要功能点**：
  - 输入与触发：
    - 按 Enter 触发搜索，或输入节流搜索。
  - 加载状态：搜索请求中显示 loading。
  - 结果点击跳转详情页。
- **建议组件**：
  - 页面：`SearchPage`
  - 区域：`SearchBar`、`SearchResultList`
  - UI：`SearchResultItem`

## 4. 公共布局与组件设计

### 4.1 布局组件

- `AppLayout`：
  - 负责全站的基础布局：包含 `Header`、主内容容器、`Footer`。
  - 负责主题切换逻辑（读取/保存主题偏好，切换 Tailwind 主题类）。
- `MainContent`、`Sidebar`：
  - 在桌面端组成两栏/三栏布局。
  - 在移动端自动变为单列，侧边栏下沉或折叠。

### 4.2 通用 UI 组件

- 导航相关：
  - `Header`：站点 Logo、导航菜单、搜索入口、主题切换、移动端菜单按钮。
  - `NavMenu`：移动端抽屉式菜单。
  - `Footer`：版权、备案、RSS、社交链接。
- 内容卡片：
  - `PostCard`：用于首页、分类页、标签页的文章展示。
  - `NoteCard`：用于笔记列表。
  - `WorkCard`：用于作品展示。
- 信息展示与状态：
  - `SectionTitle`：分区标题。
  - `EmptyState`：无内容时展示占位说明。
  - `LoadingSpinner` 或骨架屏组件：列表/详情加载中状态。
  - `TagBadge`、`CategoryLink`：标签和分类展示组件。
- 交互表单：
  - 通用 `Input`、`Textarea`、`Button` 组件，用于留言表单、搜索框等。
  - `CommentSection`：统一封装评论区域占位/集成第三方组件的位置。

## 5. 响应式与主题策略

### 5.1 断点与布局

- 参考 Tailwind 默认断点：`sm`（手机）、`md`（平板）、`lg`（桌面）、`xl`（大屏）。
- 移动端（< `md`）：
  - 顶部导航收敛为 Logo + 汉堡菜单，菜单内容在抽屉中展示。
  - 侧边栏内容（如热门文章、标签云、TOC）折叠为可展开区域或移动到底部。
- 桌面端（≥ `lg`）：
  - 支持两栏布局（正文 + 侧边栏），文章详情页 TOC 固定在侧边栏。

### 5.2 主题切换

- 明暗主题：
  - 使用全局类（如 `dark`）切换 Tailwind 风格。
  - 自定义 CSS 变量用于背景色、文字色、卡片色等。
- 用户偏好：
  - 默认跟随系统主题（`prefers-color-scheme`）。
  - 用户主动切换后，使用 localStorage 记住选择。

## 6. 数据展示与状态管理原则

- 列表页面（首页、笔记、分类、标签、作品、留言列表等）：
  - 首次加载：展示骨架屏或 loading 提示。
  - 空数据：使用 `EmptyState` 提示“暂无内容”，不留空白。
  - 错误：展示错误提示组件，可引导用户刷新重试。
- 详情页面（文章、笔记、作品）：
  - 加载中：主内容区域 skeleton。
  - 未找到：展示 404 风格的“内容不存在”提示与返回首页按钮。
- 提交表单（留言、搜索等）：
  - 客户端基础校验后再发起请求。
  - 提交中禁用按钮，避免重复提交。
  - 成功后给予明确反馈（如 toast/提示文案）。
