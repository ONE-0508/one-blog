# 个人博客网站 · 产品需求文档（PRD）

**文档类型**：产品需求文档（用于产品与界面设计参考）  
**版本**：V1.1  
**更新日期**：2026年3月  
**说明**：本文档以产品与体验为主，定义功能范围、页面结构与非功能要求；并补充少量「实现边界/约束」用于对齐前后台分工。

---

## 一、文档概述

### 1.1 项目简介

个人博客网站是一款面向写作者与知识分享者的内容发布与展示平台。产品定位为「简洁、高效、可定制」的个人内容中心，支持**笔记记录**、**博客文章发布**、**留言板互动**、**作品展示**与**数据统计**，并具备完善的**移动端适配**能力，兼顾阅读体验与轻量运维。

### 1.2 产品目标

| 目标类型 | 描述 |
|---------|------|
| 核心目标 | 让作者能快速发布、管理文章，读者能舒适阅读与互动 |
| 体验目标 | 支持明暗主题、响应式布局、无障碍与性能优化；移动端优先保证阅读与操作 |
| 扩展目标 | 预留 RSS、评论/留言、统计等能力，便于后续迭代 |

### 1.3 实现边界与约束（用于对齐分工）

- **前台**：需支持 **笔记的记录、博客文章、留言板块、作品展示、数据统计、移动端适配**。  
- **后台管理端**：可选用现成的后台管理 UI 框架/模板进行二次开发（如 Admin 端脚手架、组件库等）。  
- **后端服务**：需**手写/自研**（包括但不限于鉴权、业务接口、数据存储与管理后台所需 API），不以“一键安装的现成博客系统”作为后端主体。  

### 1.4 参考市场产品（2024–2025 趋势摘要）

- **VanBlog**：Markdown、图床、流量统计、暗色模式、TOC、草稿管理  
- **ZrLog**：插件/主题、PWA、AI 助手、所见即所得编辑  
- **Substack / Medium**：订阅、邮件通知、简约阅读体验  
- **WordPress / 静态博客**：分类/标签、SEO、多端适配  

下文功能设计在以上趋势基础上做了收敛与优先级划分。

## 二、用户角色与使用场景

### 2.1 用户角色

| 角色 | 描述 | 主要诉求 |
|------|------|----------|
| **内容作者（博主）** | 网站所有者，发布与管理内容 | 便捷写作、分类管理、数据查看、基础设置 |
| **读者/访客** | 浏览与阅读内容的用户 | 快速找到文章/笔记/作品、舒适阅读、留言互动或订阅更新 |

### 2.2 典型使用场景

- **作者**：登录后台 → 写/编辑文章或笔记（支持 Markdown）→ 设置分类/标签 → 发布或存草稿 → 查看访问统计  
- **读者**：打开首页/分类/标签/搜索 → 阅读文章/笔记（目录、代码高亮）→ 在留言板留言或通过 RSS/邮件订阅  
- **读者**：浏览作品展示 → 了解项目/作品 → 跳转到仓库/演示地址（若有）  
- **作者**：在设置中配置站点信息、主题、留言/评论、统计等

## 四、页面结构（供界面与信息架构设计）

### 4.1 前台（读者端）页面

| 页面 | 路径建议 | 核心内容与元素 |
|------|----------|----------------|
| **首页** | `/` | 站点名称/Logo、导航栏、文章列表（卡片或列表）、分页、页脚（版权、RSS、关于链接） |
| **文章详情** | `/post/:slug` 或 `/p/:id` | 标题、作者、发布时间、分类/标签、正文、TOC 侧栏/顶部、代码高亮、上一篇/下一篇、评论区、分享/复制链接 |
| **分类列表** | `/category/:name` | 分类名称、该分类下文章列表、分页 |
| **标签列表** | `/tag/:name` | 同上，按标签 |
| **归档** | `/archives` | 按年/月分组展示文章标题与日期 |
| **关于** | `/about` | 个人介绍、头像、社交链接、站点说明 |
| **友链** | `/links`（可选） | 友链列表（名称、链接、简介） |
| **搜索** | `/search` 或弹窗 | 搜索框、关键词、结果列表（标题、摘要、日期） |

### 4.2 后台（作者端）页面

| 页面 | 路径建议 | 核心内容与元素 |
|------|----------|----------------|
| **登录** | `/admin/login` | 账号/邮箱、密码、登录按钮、记住登录（可选） |
| **仪表盘** | `/admin` 或 `/admin/dashboard` | 总文章数、草稿数、近期访问趋势或简单统计、快捷入口（写文章、管理分类） |
| **文章列表** | `/admin/posts` | 表格：标题、分类、状态、发布时间、操作（编辑/删除）；筛选：状态/分类/关键词；新建按钮 |
| **编辑/发布文章** | `/admin/posts/new`、`/admin/posts/:id/edit` | 标题、正文（Markdown 编辑器+预览）、分类/标签选择、封面图、摘要、发布/存草稿/定时发布 |
| **分类管理** | `/admin/categories` | 分类列表、新增/编辑/删除、排序（可选） |
| **标签管理** | `/admin/tags` | 同上 |
| **评论管理**（若有） | `/admin/comments` | 评论列表、审核/回复/删除 |
| **设置** | `/admin/settings` | 站点名称、描述、Logo、主题（明/暗/跟随系统）、评论开关、统计代码、SEO 基础项 |

### 4.3 全局与导航

- **前台导航**：首页、分类（下拉或独立页）、标签、归档、关于、搜索入口、RSS 图标。  
- **后台导航**：仪表盘、文章、分类、标签、评论（若有）、设置。  
- **通用**：页脚可放版权、备案号、RSS、关于、友链、主题切换。

## 五、功能说明与设计要点（供交互与界面设计）

### 5.1 内容管理

- **文章列表（后台）**：支持按状态（已发布/草稿）、分类、时间、关键词筛选；列表展示标题、分类、状态、发布时间、操作；支持批量删除（可选）。  
- **文章编辑**：  
  - 标题必填，长度限制（如 1–100 字）。  
  - 正文支持 Markdown，可选所见即所得或分屏预览。  
  - 分类单选，标签多选（可输入创建新标签）。  
  - 摘要可选填，不填则自动截取正文前 N 字。  
  - 封面图可选，用于列表与分享预览。  
- **草稿与定时发布**：保存草稿不展示在前台；定时发布需选择日期时间，到期自动发布（需在需求中注明是否首版实现）。  

**界面提示**：编辑页需明确「发布」「存草稿」「定时发布」按钮状态与二次确认（如定时发布、覆盖已有文章）。

### 5.2 站点展示（前台）

- **首页**：  
  - 文章列表展示：标题、摘要、封面（可选）、分类/标签、日期。  
  - 分页或「加载更多」，需约定每页条数（如 10/20）。  
- **文章详情**：  
  - 正文排版：字号、行高、段落间距需适合长文阅读。  
  - 代码块：指定代码高亮主题（明暗主题各一套）。  
  - 目录（TOC）：根据标题层级生成，可固定在侧边或顶部，高亮当前段落。  
- **分类/标签页**：左侧或顶部为分类/标签列表，右侧为对应文章列表；或先选分类/标签再进入列表页。  
- **归档**：按年→月折叠或平铺，点击展开某月文章列表。  

**界面提示**：列表与详情需统一卡片风格、圆角、阴影等，便于做 UI 规范。

### 5.3 阅读体验

- **明暗主题**：至少支持亮色/暗色两种主题；可选「跟随系统」。切换入口固定（如导航栏或页脚图标）。  
- **响应式**：  
  - 断点建议：移动端（如 &lt;768px）、平板（768–1024px）、桌面（&gt;1024px）。  
  - 移动端：导航可收起到汉堡菜单；TOC 可折叠为抽屉或顶部下拉。  
- **无障碍**：  
  - 按钮与链接具备可读的焦点状态。  
  - 图片需支持 alt 文案（后台编辑时提醒填写）。  
  - 正文与背景对比度符合 WCAG 2.1 AA（建议 4.5:1 以上）。  

**界面提示**：设计稿中需分别输出亮色/暗色两套配色与关键组件状态（默认、hover、focus）。

### 5.4 互动与订阅

- **评论**：  
  - 展示在文章详情底部；支持昵称、邮箱、内容；可选是否需要审核后显示。  
  - 若首版采用第三方评论，需在 PRD 中注明「评论区仅预留位置与样式，对接 Gitalk/Disqus/自建等」。  
- **RSS**：  
  - 提供 RSS 2.0 或 Atom 订阅地址（如 `/rss.xml` 或 `/feed`）。  
  - 在页面 &lt;head&gt; 或页脚提供 RSS 链接，便于订阅器发现。  
- **站内搜索**：  
  - 输入关键词后展示标题、摘要、链接、日期；可列表或卡片展示。  
  - 无结果时明确提示「未找到相关文章」。  

**界面提示**：评论表单字段、错误提示、成功提示需在原型中标注。

### 5.5 后台与数据

- **登录**：  
  - 仅支持账号密码时：错误提示「账号或密码错误」，不区分具体项。  
  - 可选「记住我」延长登录态。  
- **仪表盘**：  
  - 卡片：文章总数、草稿数、评论数（若有）。  
  - 简单访问趋势：按日或按周的 PV/UV（若首版无埋点，可预留占位）。  
- **基础统计**：  
  - 若首版实现：文章详情页可展示「阅读数」；后台可展示每篇文章阅读数。  
  - 若采用第三方统计，需在设置页预留「统计代码」配置项。  

**界面提示**：仪表盘数字与图表的空状态、加载状态需在 UI 规范中说明。

### 5.6 扩展与运维（与界面相关的部分）

- **图片/图床**：  
  - 编辑器中支持粘贴图片或点击上传；上传后插入 Markdown 图片语法。  
  - 若使用图床，设置页可有「图床配置」区域（地址、密钥等），仅管理员可见。  
- **SEO**：  
  - 设置页：站点名称、一句话描述、关键词（可选）。  
  - 文章编辑：每篇文章可单独设置 SEO 标题与描述；不填则用文章标题与摘要。  
  - 前台需预留 sitemap、robots 的说明（具体实现落在开发文档）。  

**界面提示**：设置页按「基础信息」「外观」「评论」「统计」「SEO」「图床」等分组，便于 Tab 或分组表单设计。

## 六、非功能需求（与界面/体验相关部分）

### 6.1 性能与加载

- 首屏关键内容优先加载；图片建议懒加载。  
- 列表与详情在弱网下应有加载中状态（骨架屏或 Loading）。  
- 目标：在常规 3G/4G 下，首屏可交互时间 &lt; 3 秒（可作为设计评审时的体验目标）。

### 6.2 兼容与适配

- 浏览器：Chrome、Safari、Firefox、Edge 近两版；移动端 iOS Safari、Android Chrome。  
- 分辨率：支持 320px 至 1920px 宽度；大屏下内容最大宽度可限制（如 1200px）以保证可读性。

### 6.3 安全与权限

- 后台所有接口需登录后访问；未登录访问后台跳转登录页。  
- 敏感操作（删除文章、修改设置）可考虑二次确认弹窗（在交互说明中注明）。

### 6.4 文案与多语言（可选）

- 首版可仅支持中文；若后续支持英文，需在组件与文案设计时预留「键值」或多语言占位。  
- 错误提示、空状态、成功提示需统一文案风格（可单独列一份「文案规范」供设计使用）。

## 七、界面设计建议（供 UI 设计参考）

### 7.1 信息层级

- **前台**：导航 &gt; 文章标题/列表 &gt; 正文/摘要 &gt; 分类/标签/日期 &gt; 评论/次要信息。  
- **后台**：主导航 &gt; 页面标题与操作按钮 &gt; 列表/表单 &gt; 辅助说明。

### 7.2 视觉风格建议

- 以阅读为主，避免过多装饰；留白和字号层次比装饰更重要。  
- 正文字号建议 16–18px，行高 1.6–1.8；代码块字号可略小（14px）并配等宽字体。  
- 暗色模式：背景避免纯黑，使用深灰（如 #121212）；文字与背景对比度需达标。

### 7.3 组件与状态

建议在设计系统中统一约定：

- 按钮：主要 / 次要 / 文字按钮；默认、hover、loading、disabled。  
- 输入框：默认、focus、错误、禁用。  
- 卡片：列表卡片、详情内容区、统计卡片。  
- 空状态：无文章、无评论、搜索无结果、无数据。  
- 加载状态：列表骨架屏、按钮 loading、整页 loading。

### 7.4 关键流程的界面表达

- **发布流程**：编辑 → 存草稿 / 发布 / 定时发布 → 成功提示 → 跳转列表或继续编辑。  
- **阅读流程**：首页/分类/标签/搜索 → 点击文章 → 正文 + TOC + 评论；移动端 TOC 可折叠。  
- **设置流程**：分组表单 → 保存 → 成功/失败提示；敏感项可加「恢复默认」或说明文案。
